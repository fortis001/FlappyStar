# FlappyStar

## 1. 개요
Unity로 제작한 플래피 버드 스타일의 미니 프로젝트입니다.<br>
프로젝트 자체 로직보다는 코어 패키지로 최대한 빠르게 완성하는 것을 목표로 잡았습니다.

## 5. 주요 기능
### 5.1 장애물 조립 및 생성
장애물 부품이 서로 다른 스프라이트와 내부 개체를 가진 배리언트 프리팹으로 구성되어 있어, 다중 프리팹 풀을 이용해 풀링 후 조립한다.
- 플레이어의 크기와 여윳값을 기준으로 통과 가능 영역 계산
- 진행 시간에 따라 여윳값 스케일링으로 난이도 조정
- 중간의 통과 가능한 영역 생성
- 장애물 부품 프리팹 배리언트를 뽑아 조립
- 오브젝트 풀링으로 지나간 장애물 분해 후 반환
### 5.2 커스텀 타임 매니저와 게임 오버 연출
일시중지 기능과 게임 오버 시의 연출에서 멈춰야 하는 것과 그렇지 않은 부분을 구분하기 위해 타임 채널과 커스텀 타임 매니저를 다듬었다.<br>
또한 씬 로드 시에 델타타임이 크게 튀는 문제를 발견했고 한 틱을 스킵하거나 델타타임 최대값을 주는 식으로 해결했다.
- 첫 프레임에 델타타임 튀는 문제 해결
- 만들어진 타임매니저를 통해 간단한 게임오버 연출 구현
  
## 6. 아키텍처
### 6.1 오브젝트 풀링 패키지화
반복되는 오브젝트 풀링 패턴을 쉽게 가져다 쓸 수 있도록 유틸 패키지에 풀링 패턴을 추가했다.
- 싱글 프리팹 풀과 배리언트 프리팹 풀로 나뉨
- 배리언트 풀은 프리팹별로 딕셔너리를 만들어 추적
- OnGet, OnRelease, OnCreate로 인터페이스 구현
- 내부적으로 자동 호출

## 8. 고민했던 부분들
이번 프로젝트는 만들어진 패키지를 이용해 간단한 기획을 얼마나 빨리 구현할 수 있는지<br>
스스로 타임어택에 도전해보고자 하였습니다.<br>
첫 목표는 1일만에 완성이었으나 중간에 버그를 해결하는 데에 시간을 많이 잡아먹혀<br>
결국 완성까지는 2일이 걸렸던 프로젝트입니다.
- 인게임에서만 인풋이 안 먹히는 버그 : 테스트 싱글톤들을 지우지 않았다.
- 장애물 쌍의 위치가 제멋대로 나오는 버그 : 받은 스프라이트 에셋의 피봇값이 제멋대로였다.
- 시작 시 아래쪽의 방청객과 팜플렛 등의 꾸밈 요소가 이상하게 나오는 버그 : 델타타임이 튀었던 버그라 타임매니저를 수정했다.<br>
위처럼 버그 유형에는 코드 로직만 있는 것이 아니라는 것을 깨달았기 때문에 유의미한 경험이었다 생각합니다.
